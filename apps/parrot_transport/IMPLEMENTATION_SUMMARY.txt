## Architecture Highlights

1. **Protocol-Agnostic Design**
   - Zero SIP knowledge in transport layer
   - Message contracts via typed structs
   - Handler processes receive IncomingPacket structs

2. **RFC 3261 Compliance**
   - Section 18.3 Content-Length framing
   - Handles all edge cases (partial messages, multiple messages, etc.)
   - Tested with realistic TCP chunking scenarios

3. **gen_statem State Machines**
   - Explicit state management with :state_functions callback mode
   - State enter callbacks for initialization
   - Clean state transitions with proper cleanup

4. **OTP Supervision**
   - DynamicSupervisor for listener processes
   - Temporary restart strategy (listeners don't auto-restart)
   - Proper error propagation from supervisor chain

5. **Connection Management**
   - Automatic reconnection for TCP connections
   - Configurable retry logic (count, interval)
   - Socket cleanup on all state transitions

## Key Design Decisions

1. **TDD Throughout** - Every line of code has a test
2. **Synchronous Socket Binding** - Errors caught immediately in init/1
3. **Content-Length in Connection** - Framing handled per-connection, not in listener
4. **Handler Pattern** - Simple message passing, no callbacks
5. **Small Commits** - Easy to review and track changes
